CX Agent Builder
PRD in → a designed, built & evaluated CX Agent Studio agent out. Built on the official cxas-scrapi skills.

Turning a PRD into a production CX agent is manual, inconsistent, and under-tested. This is a 7-stage prompt pipeline that fixes that — model-agnostic (Claude or Gemini), delegating on-platform work to Google's official cxas-scrapi skills.

Interview gate

It asks the blocking questions and won't design until they're answered. No building on guesses.

Dual output

The same agent as a console runbook and real cxas JSON config — UI and CI/CD stay in sync.

Test-first

All 5 eval types with KPI thresholds; every requirement traces to a test.

Debug loop

Every bug becomes a documented root cause + a permanent regression test.

Grounded

Facts route through Data Stores / tools; guardrails on every agent.

Complexity-tiered

Simple bots get a lean path; complex voice/multi-agent gets full rigor.

Real CX Agent Studio format (matches the cxas-agent-foundry project template)

<project>/ cxas_app/<App>/ # the pushable app — JSON, one folder per resource app.json # displayName, rootAgent, variableDeclarations, thresholds agents/<Agent>/<Agent>.json # instruction:"…/instruction.txt", tools:[…], childAgents:[…] agents/<Agent>/instruction.txt # playbook instructions live here, not inline tools/<tool>/<tool>.json # pythonFunction | openApiTool | dataStoreTool evals/ # eval AUTHORING — YAML (Stage 5) goldens/*.yaml simulations/*.yaml callback_tests/…

Sub-agents = childAgents. Variables live in app.json. guardrails/ + toolsets/ are valid folders. To deploy: cxas create → apps list → push → push-eval. Verify with smoke-test.py --pull-dir cxas_app/<App>.
